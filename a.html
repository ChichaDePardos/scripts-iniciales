Guía Definitiva: Generador Masivo de Certificados

Sigue los pasos uno a uno. No necesitas saber programar, solo copiar y pegar donde se indica.

Fase 1: Preparación y Lectura de Archivos

Aquí le diremos al flujo de dónde sacar la información (los Excel de Traducciones y Datos).

1. Crear el disparador Manually trigger a flow

Ve a Power Automate, crea un "Instant cloud flow" y selecciona Manually trigger a flow. No agregues ninguna entrada (input). Solo dale a "New step".

2. Buscar el Excel de Traducciones List folder -> Filter array -> Get tables

Agrega estas 3 acciones en orden para encontrar dinámicamente la tabla:

  • SharePoint - List folder: Selecciona tu sitio y la carpeta de traducciones.
  • Data Operation - Filter array: En "From" pon la salida del paso anterior. En la condición pon item()?['Name'] ends with .xlsx.
  • Excel - Get tables: En File pon @first(body('Filter_array'))?['Id'].
3. Leer Traducciones List rows present in a table

Configura así:

  • File: @first(body('Filter_array'))?['Id']
  • Table: @first(outputs('Get_tables')?['body/value'])?['id']
¡MUY IMPORTANTE! Haz clic en los 3 puntitos de esta acción, elige Settings. Enciende Pagination y escribe 100000 en Threshold. Dale a Done.

* Renombra esta acción a: List_rows_Translations

4. Leer Datos Académicos (El Excel Gigante)

Repite exactamente los mismos pasos del 2 y 3, pero apuntando a la carpeta de Datos Académicos.

* Renombra la última acción a: List_rows_Datos (No olvides activar la paginación con 100000 aquí también).

Fase 2: Obtener a los 700 Alumnos Únicos

El Excel tiene miles de filas. Aquí las filtramos para que el flujo sepa cuántos certificados debe hacer.

1. Aislar los códigos de alumno Select

Agrega una acción Select (Data Operation).

  • From: Salida "value" de List_rows_Datos.
  • Haga clic en el botón pequeño a la derecha de "Map" para cambiar a modo texto y pega esta expresión exacta en la caja vacía:
@item()?['COD_ALUMNO']

* Renombra a: Select_All_Alumnos

2. Quitar duplicados Compose

Agrega un Compose. Haz clic en la caja de "Inputs", ve a la pestaña "Expression" (Expresión) en la ventana emergente, pega esto y dale a OK:

@union(body('Select_All_Alumnos'), body('Select_All_Alumnos'))

* Renombra a: Unique_Alumnos

Fase 3: Crear Variables Vacías

Agrega 12 acciones Initialize variable, una tras otra. Todas deben ser de tipo String. Déjalas todas vacías en la casilla de "Value".

  • varCodAlumno
  • varNombreAlumno
  • varModalidad
  • varUltimoPeriodo
  • varHTML
  • varCiclo
  • varEnrolled
  • varPass
  • varPondActual
  • varPondAcum
  • varOrden
  • varMerito

Fase 4: El Bucle Maestro (La Fábrica de Certificados)

Aquí es donde ocurre la magia. Procesaremos alumno por alumno.

1. Crear el Bucle Apply to each

Agrega un Apply to each. En la caja, selecciona la salida del paso Unique_Alumnos. Renómbralo a Apply_to_each_Alumno.

⚠️ REGLA DE ORO (Si no haces esto, los datos se mezclarán):
1. Clic en los 3 puntos del Apply_to_each_Alumno.
2. Selecciona Settings.
3. Enciende Concurrency Control.
4. Desliza la barra hasta el número 1. Dale a Done.

A partir de aquí, TODO va DENTRO del Bucle "Apply_to_each_Alumno"

2. Filtrar filas de este alumno Filter array

From: Salida "value" de List_rows_Datos.

Condición: item()?['COD_ALUMNO'] is equal to items('Apply_to_each_Alumno')

* Renombra a: Filter_Datos_Alumno_Actual

3. Llenar los datos del alumno Set variable

Agrega acciones Set variable con estas expresiones exactas:

varCodAlumno@items('Apply_to_each_Alumno')
varNombreAlumno@first(body('Filter_Datos_Alumno_Actual'))?['NOMBRES Y APELLIDOS']
varModalidad@first(body('Filter_Datos_Alumno_Actual'))?['COD_MODALIDAD_ESTUDIO']
varUltimoPeriodo(Déjalo totalmente vacío)

  2. Compose (Llámalo Calc_CourseName): @if(greater(length(body('Lookup_Traduccion')),0), first(body('Lookup_Traduccion'))?['COURSE'], items('Apply_to_each_Curso')?['DES_CURSO'])
  3. Compose (Llámalo Calc_UA): @if(greater(length(body('Lookup_Traduccion')),0), first(body('Lookup_Traduccion'))?['UA'], '')
